# Дашборд агента

Визуальная карта состояния агента в реальном времени: граф модулей, очередь задач, аудит.

## Как запустить

1. В `.env` добавь: `DASHBOARD=1`
2. Запусти бота как обычно: `python src/main.py`
3. Открой в браузере: **http://127.0.0.1:8765/dashboard/**

Порт можно сменить: `DASHBOARD_PORT=9000` в `.env`.

## Как это устроено

| Часть | Реализация |
|-------|------------|
| **Frontend** | Один HTML-файл (`index.html`), CSS, JavaScript, библиотека D3.js (граф с узлами и связями). |
| **Backend** | Модуль `src/hitl/dashboard_server.py`: FastAPI, эндпоинты отдают данные агента (state, queue, audit, workspace). Сервер запускается в отдельном потоке рядом с Telegram-ботом. |
| **Live update** | Фронт каждые 2 секунды запрашивает `GET /api/dashboard` и перерисовывает граф (polling). Карта обновляется по мере действий агента. |
| **Интерактивность** | **Клик** по узлу — в правой панели показываются детали (очередь, аудит, state, метрики). **Hover** — всплывающая подсказка с названием и типом узла. **Drag** узлов в D3 по умолчанию включён (можно перетаскивать узлы графа). |

## API (для своих скриптов)

- `GET /api/dashboard` — всё сразу: state, queue, audit, metrics, workspace, nodes, edges.
- `GET /api/state` — состояние агента.
- `GET /api/queue` — размер очереди и срез задач.
- `GET /api/audit?n=50` — последние записи аудита.
- `GET /api/workspace?depth=3&entries=200` — дерево файлов проекта.

Зависимости: в `requirements.txt` добавлены `fastapi` и `uvicorn`.
