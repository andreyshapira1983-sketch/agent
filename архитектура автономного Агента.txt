12. Cross-Cutting: Data Models, Flows & Contracts
   12.1 Core Data Models
   - Task (id, parent_goal_id, description, status, priority, deadline, dependencies, owner_agent, created_at)
      - producer: §3 Cognitive Core (Task Decomposition)
      - consumers: §3 (Executor), §6 (Task Queue, Scheduler)
   - Goal (id, description, success_criteria, parent_goal_id, status, priority, deadline)
      - producer: §3 (Goal Analysis & Intent Extraction, sub-goal decomposition)
      - source: §1 (User Command / intent signal)
      - consumers: §3 (Planner, Goal Conflict Resolution), §8 (Evaluation success criteria)
   - Plan (id, goal_id, steps, version, status, strategy, created_at)
      - producer: §3 (Planner)
      - consumers: §3 (Executor, Verifier), §6 (Orchestration), §8 (Evaluation)
   - Plan Step (id, plan_id, order, action_spec, preconditions, expected_outcome, status, retry_count)
      - producer: §3 (Planner)
      - consumers: §3 (Executor), §5 (Tool Layer), §6 (Retry/Backoff)
   - Action (id, step_id, type, tool_name, parameters, side_effects, timestamp)
      - producer: §3 (Executor)
      - consumers: §5 (Tool Layer), §7 (Policy gate, Action Risk classification)
   - Observation (id, source, modality, content, confidence, timestamp, provenance)
      - producer: §2 (Perception)
      - consumers: §3 (Interpret phase), §4 (Memory Write)
   - Tool Call (id, action_id, tool_name, arguments, started_at, timeout, idempotency_key)
      - producer: §3 (Act phase), §5 (Tool Adapter)
      - consumers: §5 (Tool runtime, Egress Control), §8 (Cost Tracking)
   - Tool Result (id, tool_call_id, status, output, error, latency_ms, cost, completed_at)
      - producer: §5 (Tool runtime)
      - consumers: §3 (Verifier), §5 (Tool Result Validation), §4 (Memory Write), §8 (Hallucination Detection)
   - Memory Record (id, type, content, embedding, source, confidence, ttl, provenance, created_at)
      - producer: §3 (writes from cycle), §4 (consolidation)
      - consumers: §3 (Retrieval into context), §4 (Memory Retrieval Policy)
   - User Profile (id, name, preferences, language, timezone, consent_flags, trust_level)
      - producer: §1 (UX capture), §9 (Preference Learning)
      - consumers: §1 (Personalization), §3 (User Mental Model, Planning), §7 (Consent, Permission)
   - Policy Decision (id, policy_id, subject, action, decision, reasons, timestamp)
      - producer: §7 (Governance / Policy)
      - consumers: §5 (Tool gate), §3 (Plan validation), §1 (UX disclosure)
   - Approval Request (id, action_id, requester, approver, status, justification, expires_at)
      - producer: §7 (Human Approval, Escalation)
      - consumers: §1 (UX surface), §7 (Escalation audit log)
   - Error Object (id, source, code, message, severity, recoverable, context, stack_trace, timestamp)
      - producer: all domains (on failure)
      - consumers: §6 (Recovery, Failure Taxonomy), §9 (Reflection, Self-Repair), §1 (User-Facing Audit Trail)
   - Evaluation Result (id, benchmark_id, metrics, score, baseline, regression_flag, timestamp)
      - producer: §8 (Evaluation & Benchmarking)
      - consumers: §9 (Learning, RLHF Dataset), §3 (Calibration update)
   12.2 Model Relationships
      - Goal 1..* Task
      - Goal 1..1 Plan
      - Plan 1..* Plan Step
      - Plan Step 1..* Action
      - Action 1..1 Tool Call
      - Tool Call 1..1 Tool Result
      - Observation -> Memory Record
      - Action -> Policy Decision (pre-execution gate)
      - Policy Decision -> Approval Request (when escalated)
      - Tool Result / Plan Step -> Error Object (on failure)
      - Plan / Plan Step -> Evaluation Result (post-execution)
   12.3 Core Data Flows
      - Happy Path
         - User Input (§1) -> Observation (§2)
         - Observation -> Cognitive Core Interpret (§3) -> Memory Write (§4, gated)
         - Goal extraction (§3) -> Plan / Plan Step (§3 Planner)
         - Plan Step -> Action (§3 Executor) -> Policy Decision gate (§7) -> Tool Call (§5)
         - Tool Call -> Tool Result (§5) -> Tool Result Validation (§5)
         - Validated Tool Result -> Verify (§3) -> Evaluation Result (§8)
         - Verify (success) -> Memory Write (§4) -> Output (§1) per Output Contract
      - Error Branches
         - Schema fail at §2 -> Error Object -> §1 Clarification or §6 Recovery
         - Policy denied at §7 -> Approval Request -> §1 (UX) -> human decision -> resume or abort
         - Action Risk = irreversible -> mandatory Approval (§7) before Act
         - Tool error at §5 -> Error Object -> §6 Retry/Backoff -> if exhausted -> Failure Taxonomy
         - Tool Result Validation fail (§5) -> Replan back-edge to §3
         - Hallucination Detection flag (§8) -> Replan or Escalation (§7)
         - Verify fail at §3 -> Replan or Escalation (§7)
         - Budget exhausted (§6 Runtime Budget Enforcement) -> Exit Condition or Escalation
         - Out-of-Domain detected (§7 ODD) -> Refusal + User-Facing Audit Trail
      - Back-Edges
         - Verify -> Replan (within Control Loop §3)
         - Replan -> updated Plan (§3 Planner)
         - Reflection (§9) -> Skill Library (§9) -> richer Plan templates (§3)
         - Evaluation Result (§8) -> Calibration Metrics (§8) -> Cognitive Core priors update (§3)
         - User Feedback (§1 HITL) -> Preference Learning (§9) -> User Profile (§1)
         - Failure Taxonomy (§6) -> Self-Repair (§9) -> Plan template fix (§3)
         - Memory Conflict (§4) -> Source Conflict Resolution (§4) -> Fact-Checking (§4)
      - Conditional Flows
         - if confidence < threshold -> Clarification (§3) or Escalation (§7)
         - if action.risk == irreversible -> mandatory Approval (§7)
         - if memory.relevance > threshold -> context injection (§3) within Retrieval Budget (§4)
         - if deadline approaching -> priority bump (§3) or work-shed (§6)
         - if cost > budget -> Cost-Aware Replan (§3) or Escalation (§7)
         - if multi-source conflict -> Source Conflict Resolution (§4) before use
         - if instruction conflict -> Instruction Hierarchy resolution (§7) before Plan
         - if event preempts cycle (§6 Event Bus) -> Interrupt Handler (§3) -> rescheduling
   12.4 Policy Gates (cross-cutting checkpoints)
      - Action -> Policy Decision (§7) -> if denied -> Approval Request -> §1 (UX surface)
      - Action -> Action Risk Classification (§5) -> if irreversible / high-risk -> mandatory Approval (§7)
      - Tool Call -> Permission Control (§7) -> Egress Control (§5) -> Outbound Rate Limit (§5)
      - Tool Result -> Tool Result Validation (§5) -> Hallucination Detection (§8) -> if fail -> Replan (§3)
      - Memory Write -> Memory Write Policy (§4) -> Consent Check (§1)
      - Memory Read -> Memory Retrieval Policy (§4) -> User-Specific Separation -> Relevance Threshold
      - Output -> Output Sanitization (§1) -> Output Contract enforcement (§1)
      - Plan -> Instruction Hierarchy (§7) -> Goal Conflict Resolution (§3)
      - Cycle Iteration -> Loop Control (§3) -> Runtime Budget Enforcement (§6)
      - Out-of-Domain Request -> Operational Design Domain check (§7) -> Refusal or Escalation
   12.5 Domain Interfaces
      - Perception (§2) -> Cognitive Core (§3): Observation [sync, required; on schema fail -> Error Object -> §6 Recovery]
      - Cognitive Core (§3) -> Memory (§4): Memory Record [sync write / sync read; gated by Memory Write/Retrieval Policy; on policy deny -> dropped + audit log]
      - Cognitive Core (§3) -> Tools (§5): Action / Tool Call [sync, required; on policy deny -> Approval Request]
      - Tools (§5) -> Cognitive Core (§3): Tool Result [sync or async; required; on timeout -> Error Object -> Retry/Backoff]
      - Cognitive Core (§3) -> Verifier (§3): Plan / Plan Step + Tool Result [sync, required; on fail -> Replan back-edge]
      - Runtime (§6) -> Event Bus (§6): Event [async fan-out; lossy allowed for low-priority]
      - Event Bus (§6) -> Cognitive Core (§3): Event [async interrupt; preempts current phase via Interrupt Handler]
      - Security (§7) -> All Actions (§5): Policy Decision [sync gate, required; on deny -> halt + Approval Request]
      - Security (§7) -> Interface (§1): Approval Request [async; required for irreversible actions; on timeout -> escalate or abort]
      - Evaluation (§8) -> Learning (§9): Evaluation Result [async batch; optional; on regression flag -> alert]
      - Memory (§4) -> Cognitive Core (§3): retrieved context [sync pull; budget-limited via Retrieval Budget]
      - Platform (§10) -> Runtime (§6): config / feature flags [async push; eventually consistent; on conflict -> last-write-wins with version]
      - Interface (§1) -> Cognitive Core (§3): User Command [sync request; on ambiguous -> Clarification Policy]
      - Cognitive Core (§3) -> Interface (§1): Output + Output Contract [sync response; on Output Sanitization fail -> regenerate or refuse]
      - External API (§11) -> Runtime (§6): Webhook [async push; idempotency-keyed; on duplicate -> dedupe]
   12.6 Control Flows (who triggers whom)
      - User (§1) -> Cognitive Core (§3): new command -> start cycle
      - Event Bus (§6) -> Cognitive Core (§3): event arrival -> interrupt current phase
      - Scheduler (§6) -> Cognitive Core (§3): cron / deadline fire -> initiate cycle
      - Interrupt Handler (§3) -> Loop Orchestrator (§3): preempt -> save state -> branch
      - Cognitive Core (§3) -> Tools (§5): Act phase -> tool invocation
      - Tools (§5) -> Cognitive Core (§3): tool completion -> resume cycle at Observe Result
      - Security (§7) -> Interface (§1): policy escalation -> request human approval
      - Human (§1) -> Security (§7): approval decision -> resume or abort cycle
      - Evaluation (§8) -> Learning (§9): regression detected -> trigger retraining batch
      - Reflection (§9) -> Cognitive Core (§3): new skill available -> Plan template refresh
      - Platform (§10) -> Runtime (§6): config change -> hot reload or rolling restart
      - Runtime (§6) -> All domains: shutdown signal -> graceful drain
      - Circuit Breaker (§6) -> Tools (§5): trip -> halt outbound calls; reset -> resume
      - Budget Enforcement (§6) -> Cognitive Core (§3): threshold crossed -> Cost-Aware Replan or Stop
