# Self-Model и эталон возможностей

## Файлы

- **`self_model.json`** — текущая модель себя: системы, модули, инструменты, что отсутствует (`missing`). Агент читает его, чтобы понимать свою архитектуру.
- **`capabilities_reference.json`** — эталон: все возможные системы и модули, уровни эволюции (L1–L6). Сравнение с self_model даёт разрывы.

## Инструменты

| Инструмент | Назначение |
|------------|------------|
| **analyze_self_model** | Читает оба файла, сравнивает, возвращает отчёт: что реализовано, чего нет, что улучшить. |
| **get_improvement_plan** | Строит план улучшений по результатам анализа, ранжирует по важности. |
| **update_self_model** | Добавляет или удаляет модуль в `self_model.json` (после добавления нового кода). |
| **generate_module_skeleton** | Создаёт новый модуль из шаблона (`templates/module/`, `system/`, `agent/`). |
| **apply_patch_with_approval** | Сохраняет предложенный патч в `config/pending_patches/` для проверки человеком; не применяет автоматически. |

## Модули

- **`src/reflection/self_model_analyzer.py`** — загрузка и сравнение self_model и reference, текст отчёта.
- **`src/reflection/self_improvement_planner.py`** — план улучшений по результатам анализа.

## Шаблоны

- **`templates/module/`** — один файл модуля (например `module.py`).
- **`templates/system/`** — заготовка подсистемы.
- **`templates/agent/`** — заготовка агента.

## Безопасность (гибрид A/B/C)

- Агент может **сам** вызывать `analyze_self_model`, `get_improvement_plan`, `generate_module_skeleton`, `write_file`, `run_pytest`, `update_self_model`.
- Для рискованных изменений — **apply_patch_with_approval**: патч попадает в `pending_patches/`, человек проверяет и применяет вручную или отклоняет.
- **get_audit_log** и **rollback_config** — для контроля и отката.

## Память и самообучение (что у агента уже есть)

- **Краткосрочная память (чат)** — `src/memory/short_term.py`. Последние 20 сообщений диалога с пользователем. **Сохраняется на диск** в `data/short_term_memory.json` при каждом сообщении; после перезапуска агент снова видит историю чата.
- **Обратная связь** — `src/learning/feedback.py`. Каждый обмен (запрос → ответ) пишется в feedback. **Сохраняется на диск** в `data/feedback_log.json` (последние 500 записей). Используется для контекста (последние 5 обменов в системном промпте) и для вывода правил.
- **Долгосрочная память** — `src/memory/long_term.py`. Эпизоды по user_id. **Сохраняется на диск** в `data/long_term_memory.json` (до 200 записей на пользователя).
- **Выведенные правила** — `src/learning/rules_safety.py` и `rule_derivation.py`. Правила выводятся из feedback с низким рейтингом (инструмент **derive_rules_from_feedback** или автоматически до 2 правил за автономный цикл в **improve()**). Эти правила **подставляются в системный контекст** при каждом ответе (до 20 правил с score ≥ 0.5), чтобы агент их соблюдал.

Итого: чат и feedback персистентны; выведенные правила попадают в контекст LLM. Агент «знает», что у него есть эта функциональность.

## Типовой цикл эволюции

1. Вызвать **analyze_self_model** → увидеть разрывы.
2. Вызвать **get_improvement_plan** → получить приоритизированный список.
3. Вызвать **generate_module_skeleton** (система, имя, тип) → создать файл.
4. Дописать код через **write_file**, добавить тесты, вызвать **run_pytest**.
5. Вызвать **update_self_model** (система, имя, add) → обновить self_model.json.
6. Для сомнительных правок — **apply_patch_with_approval** и проверка человеком.
