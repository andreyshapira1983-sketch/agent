# Документация агента: короткая карта

Цель: минимальный рабочий набор документов в корне и полный архив подробных версий в `docs/archive/`.

## Рабочий минимум

1. `README.md` — запуск, режимы, команды, быстрый обзор.
2. `AGENT_ARCHITECTURE.md` — архитектурный стандарт: агент первый, LLM как генератор.
3. `AUTONOMY.md` — автономный цикл, квоты, policy, инструменты.
4. `EVOLUTION.md` — патчи, sandbox, guardrails, откаты.

## Краткие документы в docs

1. `docs/ARCHITECTURE_PLAN.md` — краткая версия плана; полный текст в архиве.
2. `docs/GENETICS.md` — краткая версия генетической модели; полный текст в архиве.
3. `docs/ROADMAP.md` — краткая версия дорожной карты; полный текст в архиве.
4. `docs/USAGE.md` — практические сценарии использования.
5. `docs/TESTING.md` — проверки, smoke/full/nightly и артефакты.

## Архив полных версий

1. `docs/archive/ARCHITECTURE_PLAN_FULL.full.md`
2. `docs/archive/GENETICS.full.md`
3. `docs/archive/ROADMAP.full.md`

## Принцип консолидации

- В корне храним только то, что нужно для ежедневной работы агента и пользователя.
- Подробные и исторические версии не удаляются, а переносятся в архив.
- Новые большие разделы добавлять сначала в архивный документ, в рабочие файлы переносить только итоговые правила/интерфейсы.
