# Практическое использование агента

Краткие рабочие сценарии: как ставить задачи и как расширять инструменты.

## Что умеет агент

- Внешние сервисы: Telegram (бот), OpenAI (LLM, TTS, Whisper).
- Опционально через MCP: Notion, Linear, Stripe, Figma, Slack, браузер.

## Как ставить задачи агенту

Пишите чёткую задачу в чат. Агент сам:

- читает нужные файлы,
- предлагает или применяет правки,
- запускает проверки,
- возвращает результат.

Примеры формулировок:

- «Добавь инструмент X, реализуй в src/tools/impl, зарегистрируй и добавь тест».
- «Исправь баг в модуле Z при условии W, проверь тестами».
- «Доработай обработку голосовых сообщений в Telegram и проверь сценарий».

## Как добавлять новый инструмент

1. Добавьте модуль в `src/tools/impl/`.
2. Реализуйте функцию инструмента.
3. Зарегистрируйте её через `register(name, schema, fn)`.
4. Подключите `register_*()` в `src/tools/__init__.py`.
5. Добавьте тесты в `tests/`.

Схема инструмента: формат OpenAI function-calling (см. `src/tools/base.tool_schema`).

## Правило качества

Для нового или изменённого модуля обязательно добавляйте тесты.
Модуль считается готовым только после успешного прогона тестов.
