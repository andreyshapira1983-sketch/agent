# Обратная связь

Как оставить предложения, замечания и идеи по проекту.

---

## Каналы

- **GitHub Issues** (если репозиторий на GitHub) — задачи с метками, обсуждение, история.
- **Форма / почта** — для тех, кто не хочет заводить аккаунт: укажите в README или на сайте ссылку на форму (Google Form, Typeform и т.п.) или контакт для писем.

Конкретные ссылки (репозиторий, форма) добавляются в README при появлении.

---

## Категории

При создании Issue или отправке отзыва по возможности укажите категорию:

| Категория       | Примеры |
|-----------------|--------|
| Документация    | Неточности в README/EVOLUTION, непонятные разделы, нехватка примеров. |
| Код             | Баги, дублирование, непонятные названия, предложения по рефакторингу. |
| Тесты           | Отсутствующие тесты, хрупкие сценарии, идеи по покрытию. |
| Архитектура     | Предложения по структуре, новым модулям, ограничениям (evolution, learning, agency). |
| Использование   | Сложность запуска, конфигурации, сценарии использования. |

---

## Шаблон отзыва

Чтобы обратная связь была полезной, можно описать:

1. **Что не так или что предложить** — кратко (1–2 предложения).
2. **Как это влияет** — на использование, разработку, поддержку.
3. **Вариант решения** (по желанию) — идея, ссылка на пример, патч.

Пример:

```text
**Проблема/предложение:** В EVOLUTION.md не описан случай, когда целевой файл не существует.

**Влияние:** Непонятно, должен ли apply_patch создавать файл или только обновлять существующий.

**Вариант:** Явно указать в документации: применяем только к существующим файлам; создание — отдельный сценарий.
```

---

## Обработка

- Обратная связь просматривается периодически (например, раз в месяц/квартал).
- Решения и принятые изменения по возможности отмечаются в Issues (комментарий, закрытие с отсылкой к коммиту/документу).
- Крупные изменения, сделанные по отзывам, можно кратко перечислять в CHANGELOG или в описании релизов.

Если вы готовы начать с GitHub — создайте репозиторий и добавьте в README ссылку на Issues и при необходимости этот файл (FEEDBACK.md).
