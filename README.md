# Agent: 12 systems on Python

Автономный ИИ-агент с 12 системами: ядро интеллекта, планирование, память, знания, инструменты, исполнение, коммуникация (Telegram), управление компьютером, обучение, безопасность, финансы, эволюция. Плюс слои: Agency, Tasks, Environment, Reflection, State, Runtime, Orchestration, Meta, HITL, Multi-Agent, Perception, Personality, Monitoring.

## Что делать (без программирования)

Вам **не нужно ничего писать в коде**. Только настройка раз и запуск.

1. **Файл `.env`** (в корне проекта). Откройте его и убедитесь, что есть две строки:
   - `TELEGRAM=...` — токен бота (берёте у @BotFather в Telegram).
   - `OPEN_KEY_API=...` или `OPENAI_API_KEY=...` — ключ OpenAI (для ответов и поиска по коду).

   Если хотите, чтобы бот **сам первым писал** вам (приветствие утром/днём): добавьте строку  
   `TELEGRAM_ALERTS_CHAT_ID=ваш_id`  
   (свой id можно узнать, например, через @userinfobot в Telegram).

2. **Запуск.** В терминале из папки проекта выполните:
   ```bash
   python -m src.main
   ```
   Агент будет онлайн.

3. **Общение.** Откройте Telegram, найдите своего бота и пишите ему обычными фразами. Задавайте вопросы — он отвечает. Можете спросить про код проекта («какие модули отвечают за Telegram?», «где эмоции?») — он сам использует поиск по коду, если индекс был собран.

4. **Один раз — поиск по коду (по желанию).** Чтобы агент лучше «разбирался в коде», добавьте в `.env` одну строку:
   ```
   BUILD_CODE_INDEX=1
   ```
   При следующем запуске бот сам соберёт индекс и эмбеддинги в фоне (нужен тот же ключ OpenAI). Больше ничего настраивать не нужно.

**Чтобы агент сам ставил цели и «мысли» (визор на дашборде):** добавь в `.env` строку `AUTONOMOUS_START=1` или в Telegram напиши `/autonomous`. Тогда запустится цикл планирования: цель цикла и мысль будут видны на дашборде, агент сам будет выбирать задачи (изучать код, обновлять знания, писать тебе и т.д.). Без этого он только отвечает на твои сообщения и не запускает циклы.

**Итого:** заполнили `.env` (токен + ключ OpenAI), запустили `python -m src.main`, общаетесь в Telegram. Опционально — `BUILD_CODE_INDEX=1` для поиска по коду, `AUTONOMOUS_START=1` для автономных целей и мыслей.

**Если в Telegram часто приходят «⚠️ Tool slow»** (инструмент работал дольше порога) — это не ошибка, а предупреждение. Чтобы реже их видеть (например для fetch_url и поиска книг 3–5 секунд нормально), добавьте в `.env` строку: `TOOL_SLOW_THRESHOLD_SEC=5` (или 6). Тогда алерт придёт только если инструмент работал дольше 5 секунд.

## Требования

- Python 3.10+
- Для распознавания речи в видео (что вы говорите в видеосообщении): **ffmpeg** в PATH. Без ffmpeg агент по-прежнему «видит» кадр, но не расшифрует звук. Установка: [ffmpeg.org](https://ffmpeg.org) или `winget install ffmpeg` (Windows).
- `.env` с ключами:
  - `OPEN_KEY_API` или `OPENAI_API_KEY` — ключ OpenAI (для LLM, TTS, Whisper).
  - `TELEGRAM` — токен бота в формате `id:token` или просто `token`.
  - `TELEGRAM_ALERTS_CHAT_ID` — (опционально) ID чата для алертов и автономных уведомлений. Если не задан, они только в лог и audit.
  - `TELEGRAM_ALERTS_MIN_INTERVAL_SEC` — (опционально) мин. интервал между автономными уведомлениями в сек (по умолчанию 60). `TELEGRAM_AUTONOMOUS_EVENTS=0` — отключить автономные события. Пример — в `.env.example`.

## Запуск

```bash
python -m src.main
```

или из корня проекта:

```bash
python src/main.py
```

Входящие сообщения в Telegram проходят через память (краткосрочный контекст) и ядро (OpenAI); ответ отправляется обратно в чат.

**Память и самообучение:** чат (short_term), feedback и long_term сохраняются на диск в `data/` (`short_term_memory.json`, `feedback_log.json`, `long_term_memory.json`) — после перезапуска история доступна. Выведенные из feedback правила подставляются в контекст LLM; в автономном цикле в `improve()` до 2 правил за цикл выводятся автоматически. Подробнее — в `config/SELF_MODEL.md`, раздел «Память и самообучение».

Агент **автономный**: сам решает, что делать (observe → reason → plan → act → reflect → improve). Он не ждёт команд — в автономном режиме крутит циклы сам и присылает отчёты. В чат можно писать что угодно: он отвечает и при необходимости запускает цикл (например по «Да» или «запусти» после статуса).

**Режим и мониторинг в Telegram** — не «команды управления», а способ включить автономию и посмотреть состояние:

- **`/autonomous`** — включить автономный режим: циклы идут в фоне, отчёты приходят в чат. Агент работает сам.
- **`/stop`** — остановить автономный режим (после текущего цикла).
- **`/status`** — посмотреть состояние: квоты, последний цикл, ошибки, метрики.
- **`/tasks`** или **`/queue`** — посмотреть очередь задач (что агент запланировал).
- **`/log [N] [тип] [модуль] [приоритет]`** — посмотреть последние автономные действия из audit (по умолчанию 30). Опционально: `N` (до 100), тип действия, модуль, приоритет. Примеры: `/log 50`, `/log 20 autonomous_act`.
- **`/mood`** или **`/emotions`** — посмотреть эмоциональную матрицу агента (0–1 и доминанта).

### Эмоциональная матрица агента

В `src/personality/` реализована матрица эмоций (0–1) с затуханием со временем: **frustration**, **excitement**, **anxiety**, **curiosity**, **fatigue**, **aggression**. Каждое действие или событие обновляет эмоции через триггеры (`triggers.py`): «пошёл спать» → снижает возбуждение и агрессию, повышает усталость; падение теста/ошибка инструмента → фрустрация и тревога; успешный цикл → возбуждение; и т.д. Есть случайные толчки для непредсказуемости. Ответы в Telegram и автономные уведомления могут дополняться эмоциональной окраской (⚡ возбуждён, 💀 устал, 🔥 злится и т.п.). Динамика: `emotion_value += delta`, затем затухание `emotion_value *= decay` при каждом тике. Эмоции **напрямую влияют на приоритеты и нагрузку**: (1) **Очередь задач** — при извлечении из очереди используется эмоциональная оценка (`emotion_task_priority.py`): при высокой усталости предпочитаются «лёгкие» задачи (get_metrics, get_current_time), при тревоге — перепроверка метрик, при любопытстве — исследовательские. (2) **Лимит действий в цикле** — при `sleep_mode` выполняется не более 1 действия за цикл, при `reduce_actions` — не более половины квоты. (3) **Порядок патчей** — инструмент `list_pending_patches(emotion_aware=True)` возвращает патчи в порядке, зависящем от эмоций: при высокой тревоге/фрустрации — сначала мелкие по размеру, при любопытстве — сначала новые.

### Как включить уведомления об алертах в Telegram

Чтобы агент присылал уведомления об алертах в Telegram, сделайте следующее:

1. **Определите ID чата или пользователя**, куда будут приходить уведомления:
   - **Личный чат** — числовой `chat_id` пользователя или @username (часто нужен именно числовой id; его можно узнать через @userinfobot или из логов при отправке боту сообщения).
   - **Группа** — числовой `chat_id` группы (обычно **отрицательное** число).

2. **Добавьте переменную в `.env`:**

   ```env
   TELEGRAM_ALERTS_CHAT_ID=123456789
   ```

   Замените `123456789` на свой chat_id (для группы — с минусом, например `-1001234567890`).

3. **Перезапустите агента**, чтобы он подхватил обновлённую конфигурацию.

После этого все события, помеченные как alert (например **performance_alert**, **tool_slow**, **patch_failed** / ошибка инструмента), а также автономные уведомления будут автоматически отправляться в указанный чат через Telegram.

### Автономные уведомления в чат

При заданном `TELEGRAM_ALERTS_CHAT_ID` бот присылает в этот чат сообщения об автономных действиях с эмодзи (⚠️ срочное, ✅ успех, 📝 правка):

- **✅ [Автономное действие]** — резюме цикла (цель, кол-во действий, успехи/ошибки, патчи в цикле), обновление правил промпта, предложенное изменение файла.
- **⚠️ [СРОЧНО]** — патч предложен для одобрения (без троттлинга).

Частота ограничена троттлингом: по умолчанию не чаще одного такого уведомления в 60 секунд (`TELEGRAM_ALERTS_MIN_INTERVAL_SEC`). Срочные уведомления идут без задержки. Отключить: `TELEGRAM_AUTONOMOUS_EVENTS=0`.

**Фильтр видимости шагов:** задай `TELEGRAM_EVENT_TYPES=task_start,patch,report` (или `cycle,goal,task_start,patch,report`), чтобы получать короткие сообщения: «Начал: read_file», «Применил патч», «Сгенерировал отчёт». Интервал между шагами — `TELEGRAM_STEP_INTERVAL_SEC` (по умолчанию 10 с).

**Дашборд:** блок «Фокус» показывает только важное: текущая цель, текущая задача в очереди, мысль планировщика и последние 5 действий — не нужно листать весь лог.

## Структура (кратко)

- `src/core` — ядро интеллекта, промпт, язык.
- `src/communication` — Telegram-клиент.
- `src/memory` — краткосрочная и долгосрочная память, контекст-менеджер.
- `src/planning` — планировщик, цели, симуляция.

- `src/execution` — исполнение планов и вызов инструментов.
- `src/security` — политики, песочница, этика.
- `src/finance` — учёт использования, лимиты.
- `src/learning` — обратная связь, самообучение.
- `src/evolution` — конфиги, версионирование, саморемонт.
- `src/personality` — эмоциональная матрица (emotion_matrix, triggers, emotional_reactions).
- Остальные каталоги — Agency, Tasks, State, Runtime, HITL, Multi-Agent и др. (см. план архитектуры).

## Конфиг validate_project_layout

Проверка структуры проекта (`validate_project_layout`) теперь может брать дефолты из `config/layout_validation.json`:

- `symbol_check_mode`: `strict`, `relaxed`, `off`
- `symbol_check_whitelist`: массив путей/шаблонов, где symbol-check пропускается
- `summary_path`: куда писать короткий машинно-читаемый summary JSON
- `summary_top_n`: сколько приоритетных проблем включать в summary

Если параметр передан в вызове инструмента, он имеет приоритет над значением из конфига.
Шаблон: `config/layout_validation.json.template`.

Поддерживаются профили через поля `active_profile` + `profiles`:

- `shadow` — максимально исследовательский режим
- `canary` — промежуточный
- `stable` — консервативный прод-режим

Переключение без редактирования файла:

- `AGENT_VALIDATION_PROFILE=shadow|canary|stable` — профиль для validate_project_layout
- `AGENT_PROFILE=shadow|canary|stable` — профиль для `config/agent.json` (autonomy_limits/tool_performance)
