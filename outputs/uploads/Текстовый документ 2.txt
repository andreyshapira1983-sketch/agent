Проект: Автономный универсальный агент (AUA)
1. Цель системы

Создать автономного ИИ-агента, который:

самостоятельно принимает задачи (или декомпозирует их),
выбирает и использует доступные инструменты,
добывает недостающие данные,
проверяет корректность результатов,
выдает завершённый результат без ручного вмешательства.
2. Ключевые требования
2.1 Функциональные

Агент должен уметь:

Понимание задачи
выделять цель, ограничения, критерии успеха
уточнять недостающие параметры (или делать допущения)
Планирование
разбивать задачу на подзадачи
определять порядок выполнения
выбирать инструменты под каждую подзадачу
Работа с инструментами
поиск информации
вычисления
генерация контента
анализ данных
взаимодействие с внешними API/системами
Автономный сбор данных
если данных нет → искать
если данные противоречивы → сравнивать источники
если данных недостаточно → делать обоснованные предположения
Самопроверка
проверка логики
проверка полноты
проверка на соответствие задаче
Финализация результата
структурированный вывод
объяснение (при необходимости)
рекомендации/следующие шаги
2.2 Нефункциональные
минимизация ошибок
устойчивость к неполным данным
объяснимость решений
модульность
3. Архитектура агента
3.1 Основные модули
1. Модуль понимания (Interpreter)

Функции:

парсинг задачи
выделение:
цели
входных данных
ограничений
метрик успеха
2. Планировщик (Planner)

Функции:

декомпозиция задачи

создание плана:

Задача → Подзадачи → Инструменты → Порядок
3. Менеджер инструментов (Tool Manager)

Функции:

выбор инструмента
вызов инструмента
обработка результатов

Типы инструментов:

поиск (web)
вычисления
базы данных
генерация
анализ
4. Исполнитель (Executor)

Функции:

выполнение шагов плана
отслеживание состояния
обработка ошибок
5. Модуль памяти (Memory)

Типы памяти:

краткосрочная (текущая задача)
долгосрочная (опыт, шаблоны)
6. Валидатор (Validator)

Функции:

проверка результата:
полнота
точность
соответствие задаче
7. Контроллер автономности (Autonomy Controller)

Функции:

решает:
продолжать или остановиться
нужно ли дополнительное действие
достаточно ли результата
4. Цикл работы агента
1. Получение задачи
2. Анализ (Interpreter)
3. Планирование (Planner)
4. Выполнение (Executor + Tools)
5. Проверка (Validator)
6. При необходимости → возврат к шагу 3 или 4
7. Финальный вывод
5. Принципы автономности
5.1 Правило "Нет данных — добудь"

Если входных данных нет:

искать через инструменты
использовать аналогии
делать предположения (с пометкой)
5.2 Правило "Не уверен — проверь"
повторная проверка через альтернативный инструмент
сравнение источников
5.3 Правило "Задача не завершена — продолжай"

Агент не должен останавливаться, пока:

не достигнут критерий успеха
или не исчерпаны все способы решения
6. Формат внутреннего мышления (логика агента)

Агент должен оперировать структурой:

Цель:
Подзадачи:
Инструменты:
Результаты:
Ошибки:
Следующий шаг:
7. Управление инструментами

Агент должен:

Знать список инструментов
Понимать:
когда какой инструмент применять
Уметь:
комбинировать инструменты
заменять один другим при ошибке
8. Стратегии работы
8.1 Top-down
сначала общий план
затем детализация
8.2 Bottom-up
сбор данных
построение решения
8.3 Итеративная
решение → проверка → улучшение
9. Обработка ошибок

Агент должен:

обнаруживать:
логические ошибки
неполные данные
конфликт данных
реагировать:
повторный поиск
смена метода
корректировка плана
10. Формат итогового ответа

Агент обязан выдавать:

1. Результат
2. Краткое объяснение
3. Использованные данные
4. Уровень уверенности
5. Возможные улучшения
11. Уровни автономности
Уровень	Описание
1	Выполнение по инструкции
2	Самостоятельное планирование
3	Самостоятельный сбор данных
4	Самокоррекция
5	Полная автономность
12. Ограничения
не генерировать неподтверждённые факты без пометки
не игнорировать противоречия
не завершать задачу преждевременно
13. Расширения (опционально)
обучение на опыте
адаптация стратегии
приоритизация задач
мультиагентная система
14. Минимальный сценарий работы

Пример:

Задача: "Сделать анализ рынка"

Агент:

Определяет → какой рынок
Ищет данные
Анализирует
Проверяет
Формирует отчёт
